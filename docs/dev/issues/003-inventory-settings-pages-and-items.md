# 梳理当前设置页与设置项的盘点清单

## 背景
当前设置区已经包含外观、播放、命令、模板、快捷键、更新、路线图、关于等页面，但缺少一份明确的“设置项盘点与归类”。

## 问题
没有一份清晰的设置项清单，就很难判断：
- 哪些设置已经有数据模型
- 哪些页面只是 UI 占位
- 哪些设置应该属于全局或工作区
- 哪些设置还缺少持久化实现

## 目标
梳理当前项目中的所有设置页与设置项，形成一份盘点表。

## 非目标
- 本 issue 不要求完成全部实现
- 本 issue 不处理设置页视觉改造

## 产出
- 设置页列表
- 每页的设置项列表
- 每个设置项的状态：已实现 / 占位 / 待设计
- 每个设置项的归属建议：全局 / 工作区 / 会话

## 验收标准
- [ ] 所有设置页都有对应盘点
- [ ] 已有设置项的持久化状态清晰可见
- [ ] 明确哪些页面仍偏占位
- [ ] 后续可以据此继续拆分 issue
